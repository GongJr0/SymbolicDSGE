    @dataclass(frozen=True)
class DenHaanMarcetResult()

    dataclass storing the output of a single Den Haan-Marcet (DHM) test application.

    Fields:

    Name Type Description
    statistic float Realized DHM test statistic.
    df int Degrees of freedom of the chi-square reference distribution.
    p_value float Chi-square tail probability associated with statistic.
    critical_value float Chi-square critical value implied by the requested alpha.
    rejects_null bool Whether the test rejects the null at the requested significance level.
    mean_moments np.ndarray Sample mean of the stacked moment vector.
    covariance np.ndarray Estimated covariance matrix of the stacked moment vector.
    moments np.ndarray Realized stacked moment matrix. Shape (n_obs, n_eq * n_inst).
    residuals np.ndarray Equation residuals used in the DHM test. Shape (n_obs, n_eq).
    raw_residuals np.ndarray Explicit copy of the raw residual matrix for downstream diagnostics. Shape (n_obs, n_eq).
    instruments np.ndarray Realized instrument matrix. Shape (n_obs, n_inst).
    states np.ndarray State matrix used to evaluate the test. Shape (T+1, n) for simulation-driven runs.
    shock_matrix np.ndarray \| None Realized shock matrix used by one_sample. None for from_state_path.
    variables list[str] Ordering of state columns in states. Matches compiled variable order.
    equation_idx np.ndarray Equation indices used in fallback compiled-equation mode, or a sequential index for custom FOCs.
    instrument_idx np.ndarray Instrument indices resolved against compiled variable order.
    include_constant bool Whether a constant instrument was included.
    burn_in int Number of leading transitions excluded from the test.
    foc_expressions tuple[str, ...] \| None Normalized custom FOC expressions used for the run. None in compiled-equation mode.

    @dataclass(frozen=True)
class MeasurementMomentResult()

    dataclass storing the output of a measurement residual orthogonality test.

    This return type is used by both the per-observable and joint measurement-moment interfaces. For the per-observable methods, the measurement arrays are one-dimensional. For the joint methods, they are stacked across observables.

    Fields:

    Name Type Description
    statistic float Realized chi-square test statistic.
    df int Degrees of freedom of the chi-square reference distribution after the requested df adjustment.
    p_value float Chi-square tail probability associated with statistic.
    critical_value float Chi-square critical value implied by the requested alpha.
    rejects_null bool Whether the test rejects the null at the requested significance level.
    mean_moments np.ndarray Sample mean of the measurement moment vector.
    covariance np.ndarray Estimated covariance matrix of the measurement moment vector.
    moments np.ndarray Realized moment matrix. Shape (n_obs, n_inst) for per-observable tests and (n_obs, n_inst * n_meas) for joint tests.
    observed np.ndarray Observed measurement series used in the test after burn-in and lag alignment.
    predicted_measurements np.ndarray Model-implied measurement series aligned to observed.
    measurement_errors np.ndarray Difference observed - predicted_measurements.
    instruments np.ndarray Realized instrument matrix. Shape (n_obs, n_inst).
    states np.ndarray Aligned state matrix used to construct measurements and instruments. Shape (T, n) after dropping any initial-condition row.
    shock_matrix np.ndarray \| None Realized shock matrix used by simulation-driven measurement tests. None for the *_from_state_path routes.
    variables list[str] Ordering of state columns in states. Matches compiled variable order.
    observables list[str] Observable names represented in the result. Length 1 for per-observable tests and length n_meas for joint tests.
    instrument_idx np.ndarray Instrument indices resolved against compiled variable order.
    include_constant bool Whether a constant instrument was included.
    lagged_instruments bool Whether instruments were taken from t-1 instead of t.
    burn_in int Number of leading aligned observations excluded before constructing moments.
    n_estimated_params int Number of estimated parameters subtracted from the raw moment count when forming the reported degrees of freedom.

     

    class DenHaanMarcet(
    solved: SolvedModel,
    focs: Sequence[str] | None = None,
    foc_locals: Mapping[str, str] | None = None
)

    DenHaanMarcet provides one-sample and Monte Carlo Den Haan-Marcet tests using a SolvedModel object.

    The class supports two equation sources:

    • Compiled model equations selected by equation_idx
    • Custom FOC strings supplied either at construction or per method call

    Custom FOC strings are parsed against the model configuration, validated against calibrated parameters and time-indexed variables, and normalized to a t / t-1 representation before compilation.

    Forward Object Convention

    By default, forward terms use the conditional expectations (next state if shocks were 0.0). If use_conditional_expectation=False, the forward object becomes the realized simulation draw of the next time index.

    Equation and Instrument Indexing

    In compiled-equation mode, equation_idx indexes the model equations in the compiled config.equations.model order. instrument_idx indexes the compiled variable order and can be supplied either as integer positions or variable names.

    Custom FOC Syntax

    Time-dependent variables must be written with explicit timing such as x(t) or Pi(t+1). Bare variable names like x are treated as symbols and rejected unless they are calibrated parameters or locally defined aliases.

    FOC Locals

    foc_locals allows pre-validation substitutions. Both scalar and time-indexed aliases are supported, for example:

    • {"gap_term": "sigma*(r(t) - Pi(t+1))"}
    • {"C(t)": "exp(x(t))"}

    Methods:

    DenHaanMarcet.from_state_path(
    states: np.ndarray,
    *,
    focs: Sequence[str] | None = None,
    foc_locals: Mapping[str, str] | None = None,

    instrument_idx: Sequence[int | str] | None = None,
    include_constant: bool = True,
    burn_in: int = 0,
    alpha: float = 0.05,
    use_conditional_expectation: bool = True
) -> DenHaanMarcetResult

    Run a DHM test on a user-supplied state path without simulating shocks internally.

    State Path Convention

    states must be a 2D matrix in compiled variable order. Consecutive rows are interpreted as consecutive periods. The method evaluates len(states) - 1 - burn_in transitions by pairing states[:-1] with either states[1:] or states[:-1] @ A.T.

    Inputs:

    Name Description
    states Consecutive state vectors in compiled variable order.
    focs Optional custom FOC strings used instead of compiled model equations.
    foc_locals Optional alias dictionary applied before FOC validation and compilation.
    equation_idx Optional subset of compiled-equation indices to test.
    instrument_idx Optional subset of instruments taken from compiled variable order.
    include_constant Include a constant term in the instrument vector if True.
    burn_in Number of leading transitions excluded from the DHM test.
    alpha Significance level used for critical_value and rejects_null.
    use_conditional_expectation Use E_t[X_{t+1}] = A X_t instead of the realized next row.

    Returns:

    Type Description
    DenHaanMarcetResult Single-run DHM output built from the provided state path. shock_matrix is None in this route.

    1. Monte Carlo mode requires Shock instances, not arbitrary callables or pre-realized arrays. Each replication clones the provided shocks and deterministically increments their seeds when a base seed is present.

    Run repeated DHM tests under independently regenerated shock draws.

    Monte Carlo Shock Requirements

    The Monte Carlo interface is intentionally stricter than one_sample. Repeated draws require generator-style Shock objects so the class can safely create fresh realizations without mutating user-owned shock specifications.

    DenHaanMarcet.measurement_moment_test(
    y: Mapping[str, Sequence[float] | np.ndarray] | np.ndarray,
    observable: str | Sequence[str],
    *,
    shocks: dict[str, Callable | np.ndarray] | None = None, # (1)!
    shock_scale: float = 1.0,
    x0: np.ndarray | None = None,
    instrument_idx: Sequence[int | str] | None = None, # (2)!
    include_constant: bool = True,
    lagged_instruments: bool = False, # (3)!
    burn_in: int = 0,
    alpha: float = 0.05,
    n_estimated_params: int, # (4)!
    measurement_fn: Callable[[np.ndarray, np.ndarray], np.ndarray] | None = None, # (5)!
) -> MeasurementMomentResult | list[MeasurementMomentResult]
    1. Shock specification follows the same conventions as SolvedModel.sim, including comma-separated keys for correlated multivariate shocks.
    2. Accepts either integer indices or variable names resolved against compiled variable order.
    3. If True, the test uses z_{t-1} against current measurement errors instead of contemporaneous instruments.
    4. The reported degrees of freedom are adjusted as m - n_estimated_params, where m is the number of moment conditions.
    5. If omitted, the class uses the compiled model measurement equations for the requested observables.

    Simulate a state path and run a GMM-style measurement residual orthogonality test.

    This is not a Den Haan-Marcet Euler orthogonality test. The default null is E[z_t * e_t] = 0, where e_t is the measurement residual. With lagged_instruments=True, the null becomes E[z_{t-1} * e_t] = 0.

    Degrees-of-Freedom Adjustment

    The adjustment m - p assumes the same free parameters were estimated from the same moment conditions. If that is not true, treating n_estimated_params as a df correction is heuristic.

    Observable Ordering and y Shapes

    observable may be a single observable name or a sequence. y may be:

    • a mapping keyed by observable name
    • a 1D array when exactly one observable is requested
    • a 2D array whose columns correspond to the requested observable order

    When multiple observables are requested, the returned results are ordered by the compiled observable order.

    Inputs:

    Name Description
    y Observed measurement series used to form measurement residuals.
    observable Observable name or sequence of observable names to test individually.
    shocks Shock specification used to generate the simulated state path.
    shock_scale Multiplicative scaling applied to realized shocks.
    x0 Initial state vector. None defaults to zeros with controls backfilled from the policy rule.
    instrument_idx Optional subset of instruments taken from compiled variable order.
    include_constant Include a constant term in the instrument vector if True.
    lagged_instruments Use lagged state instruments instead of contemporaneous ones.
    burn_in Number of leading aligned observations excluded from the test.
    alpha Significance level used for critical_value and rejects_null.
    n_estimated_params Number of estimated parameters subtracted from the raw moment count when forming the chi-square reference df.
    measurement_fn Optional override for the measurement mapping. It must return one value per requested observable.

    Returns:

    Type Description
    MeasurementMomentResult \| list[MeasurementMomentResult] Returns a single result when observable is a string, or one result per requested observable when a sequence is supplied.

    Run the per-observable measurement residual orthogonality test on a user-supplied state path.

    State Path Convention

    states may be either:

    • an already aligned (T, n) matrix of state vectors, or
    • a simulation-style (T+1, n) path including an initial condition row

    In the second case, the initial row is dropped so measurements align with the observation sample.

    Observable Ordering and y Shapes

    The same observable ordering and y conventions used by measurement_moment_test apply here.

    states Aligned (T, n) state matrix or simulation-style (T+1, n) path in compiled variable order.
    y Observed measurement series used to form measurement residuals.

    include_constant Include a constant term in the instrument vector if True.

    DenHaanMarcet.joint_measurement_moment_test(
    y: Mapping[str, Sequence[float] | np.ndarray] | np.ndarray,
    observables: Sequence[str] | None = None,
    *,
    shocks: dict[str, Callable | np.ndarray] | None = None,
    shock_scale: float = 1.0,
    x0: np.ndarray | None = None,
    instrument_idx: Sequence[int | str] | None = None,
    include_constant: bool = True,
    lagged_instruments: bool = False,
    burn_in: int = 0,
    alpha: float = 0.05,
    n_estimated_params: int,
    measurement_fn: Callable[[np.ndarray, np.ndarray], np.ndarray] | None = None
) -> MeasurementMomentResult

    Run a joint measurement residual orthogonality test by stacking all requested observable-error blocks into a single moment vector.

    Joint Test Construction

    The joint test uses the same instrument matrix for every requested observable and horizontally stacks the per-observable blocks z_t * e_{j,t} into one moment matrix before computing the chi-square statistic.

    Observable Selection

    If observables=None, the method uses all compiled observables in compiled order.

    Observable Ordering and y Shapes

    The same y conventions used by measurement_moment_test apply here. Requested observables are resolved into compiled observable order before the joint moment matrix is constructed.

    Inputs:

    Name Description
    y Observed measurement series used to form measurement residuals.
    observables Optional subset of observable names to include jointly. None selects all compiled observables.
    shocks Shock specification used to generate the simulated state path.
    shock_scale Multiplicative scaling applied to realized shocks.
    x0 Initial state vector. None defaults to zeros with controls backfilled from the policy rule.
    instrument_idx Optional subset of instruments taken from compiled variable order.
    include_constant Include a constant term in the instrument vector if True.
    lagged_instruments Use lagged state instruments instead of contemporaneous ones.
    burn_in Number of leading aligned observations excluded from the test.
    alpha Significance level used for critical_value and rejects_null.
    n_estimated_params Number of estimated parameters subtracted from the stacked moment count when forming the chi-square reference df.
    measurement_fn Optional override for the measurement mapping. It must return one value per requested observable.

    Returns:

    Type Description
    MeasurementMomentResult Joint result whose observables, measurement_errors, and moments are stacked across all requested observables.
